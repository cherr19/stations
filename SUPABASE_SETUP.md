# Пошаговая настройка Supabase для Founders Vision Tool

Инструкция для одного человека (например, Тани): что сделать в Supabase, чтобы приложение сохраняло ответы в облаке.

**Важно:** вы с Аленой открываете приложение **с разных устройств** (ты — с своего, она — со своего). Значит приложение должно быть доступно по **публичной ссылке в интернете**, а не только у тебя на компьютере. Поэтому после настройки Supabase (шаги 1–6) **обязательно** выкладываете приложение в интернет (деплой, шаг 7.1). Дальше вы обе работаете только по этой ссылке. Запуск «у себя на компе» (localhost) — только по желанию, чтобы проверить перед деплоем.

---

## 1. Открыть Supabase и войти

1. Открой в браузере: **[https://supabase.com](https://supabase.com)**
2. Нажми **Sign in** (Войти) в правом верхнем углу.
3. Войди через GitHub, Google или по email — в зависимости от того, как у тебя заведён аккаунт.
4. После входа ты попадёшь в **Dashboard** (панель проектов).

---

## 2. Создать новый проект

1. На главной странице Dashboard нажми зелёную кнопку **New project** (Новый проект).
2. Выбери или создай **Organization** (организацию), если спрашивают — можно оставить личную.
3. В поле **Name** введи название, например: `founders-vision`.
4. В поле **Database Password** придумай и введи надёжный пароль к базе — **сохрани его** (он понадобится только если будешь подключаться к БД напрямую; для приложения он не нужен).
5. В **Region** выбери регион (например, Frankfurt или ближайший к тебе).
6. Нажми **Create new project** внизу.
7. Дождись, пока проект создаётся (обычно 1–2 минуты). Когда статус станет зелёным — переходи к следующему шагу.

---

## 3. Создать таблицу для комнат

Проще всего создать таблицу одним запросом в SQL Editor.

1. В левом меню нажми **SQL Editor** (Редактор SQL).
2. Нажми **New query** (Новый запрос).
3. Вставь в окно запрос:

```sql
CREATE TABLE vision_rooms (
  room_id TEXT PRIMARY KEY,
  tanya_data JSONB,
  alena_data JSONB,
  updated_at TIMESTAMPTZ
);
```

1. Нажми **Run** (Выполнить) внизу справа (или Ctrl+Enter).
2. Должно появиться сообщение вроде «Success. No rows returned». Таблица `vision_rooms` создана.

**Если хочешь создать таблицу через интерфейс (Table Editor):**

1. В левом меню нажми **Table Editor** → **New table**.
2. **Name**: `vision_rooms`.
3. Удали колонку **id**, если она есть по умолчанию (или не используй её).
4. Добавь колонки по одной (**Add column**):
  - `room_id`, Type: **text**, включи **Is primary key**.
  - `tanya_data`, Type: **jsonb**, Nullable: да.
  - `alena_data`, Type: **jsonb**, Nullable: да.
  - `updated_at`, Type: **timestamptz** (или **timestamp with time zone**), Nullable: да.
5. Нажми **Save**.

---

## 4. Разрешить приложению читать и писать в таблицу

Без этого шага приложение не сможет сохранять данные: Supabase по умолчанию блокирует доступ.

Если ты зашла в **Authentication** — он для этого не нужен. В левом меню выбери **SQL Editor**.

**Что сделать:**

1. В левом меню нажми **SQL Editor** (не Authentication — политики таблицы там не настраиваются).
2. Нажми **New query** (Новый запрос).
3. Вставь в окно **ровно этот запрос**:

```sql
ALTER TABLE vision_rooms ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Allow anonymous read and write for vision_rooms"
ON vision_rooms
FOR ALL
TO anon
USING (true)
WITH CHECK (true);
```

1. Нажми **Run** (или Ctrl+Enter).
2. Внизу должно появиться что-то вроде «Success». Готово — приложение теперь может работать с таблицей.

---

## 5. Взять URL проекта и ключ

На странице **Project Settings → API** Supabase показывает только ключи. URL проекта там нет — его берёшь из адресной строки браузера.

**URL проекта**

1. Посмотри в **адресную строку браузера**. Ты на странице вида:
   `https://supabase.com/dashboard/project/evcacotmqczvsqfmgwrr/settings/api-keys`
2. Найди часть после **`/project/`** и до следующего слэша — это ID проекта (длинная строка букв).
3. URL для приложения: **`https://[ЭТОТ_ID].supabase.co`**  
   Пример: если ID = `evcacotmqczvsqfmgwrr`, то URL = `https://evcacotmqczvsqfmgwrr.supabase.co`  
   Этот адрес вписываешь в `VITE_SUPABASE_URL`.

**Ключ**

4. На той же странице скопируй ключ **Publishable** (начинается с `sb_publishable_...`). Это `VITE_SUPABASE_ANON_KEY`. Ключ **Secret** не используй.

---

## 6. Подставить данные в приложение

1. В папке проекта `founders-vision-tool` создай или открой файл **`.env.local`** (в корне, рядом с `package.json`).
2. Добавь (или замени) две строки, подставив свои значения из шага 5:
  ```
   VITE_SUPABASE_URL=https://xxxxx.supabase.co
   VITE_SUPABASE_ANON_KEY=твой_anon_ключ_без_кавычек
  ```
   Без кавычек, без пробелов вокруг `=`.
3. Сохрани файл.
4. Перезапусти приложение: в терминале останови `npm run dev` (Ctrl+C) и снова запусти `npm run dev`.
5. Открой приложение в браузере. При создании комнаты и заполнении формы данные должны уходить в Supabase. Проверь в Supabase: **Table Editor** → таблица `vision_rooms` — после сохранения ответов должны появляться строки с `room_id`, `tanya_data` или `alena_data` и `updated_at`.

---

## Краткий чек-лист

- Зашла на supabase.com и создала проект.
- В **Table Editor** создала таблицу `vision_rooms` с колонками: `room_id` (text, PK), `tanya_data` (jsonb), `alena_data` (jsonb), `updated_at` (timestamptz).
- В **SQL Editor** выполнила запрос из шага 4 (RLS + политика для `vision_rooms`).
- В **Project Settings** → **API** скопировала URL проекта и ключ **Publishable** (не Secret).
- В проекте в `.env.local` прописала `VITE_SUPABASE_URL` (URL) и `VITE_SUPABASE_ANON_KEY` (Publishable-ключ), перезапустила `npm run dev` и проверила сохранение в **Table Editor**.

## 7. Что делать дальше: деплой и работа с разных устройств

После настройки Supabase (шаги 1–6) вы с Аленой работаете **по одной публичной ссылке** с разных устройств. Нужно по порядку: (1) выложить проект в GitHub, (2) подключить его к Vercel и задеплоить. Ниже — полная последовательность без пропусков.

---

### 7.1. Выложить проект в GitHub

Без репозитория на GitHub Vercel не из чего собирать проект. Делаем по шагам.

**На сайте GitHub:**

1. Открой в браузере **[github.com](https://github.com)** и войди в свой аккаунт (или зарегистрируйся).
2. В правом верхнем углу нажми **+** → **New repository**.
3. В поле **Repository name** введи, например: `founders-vision-tool`.
4. Выбери **Private** или **Public** (на деплой это не влияет).
5. **Не ставь** галочки «Add a README file», «Add .gitignore», «Choose a license» — репозиторий должен остаться пустым.
6. Нажми **Create repository**.
7. На следующей странице GitHub покажет команды — пока не выполняй их. Скопируй **URL репозитория** (кнопка Code → HTTPS), он выглядит так: `https://github.com/ТВОЙ_ЛОГИН/founders-vision-tool.git`. Он понадобится ниже.

**На своём компьютере (Терминал):**

8. Открой **Терминал**.
9. Перейди в папку проекта:
   ```bash
   cd ~/cursor/founders-vision-tool
   ```
   (Если проект лежит в другом месте — подставь свой путь.)
10. Инициализируй git в этой папке (если ещё не инициализирован):
    ```bash
    git init
    ```
    Если увидишь «Reinitialized existing Git repository» — ничего страшного, иди дальше.
11. Добавь все файлы в коммит:
    ```bash
    git add .
    ```
12. Сделай первый коммит:
    ```bash
    git commit -m "Initial: Founders Vision Tool"
    ```
13. Укажи ветку `main` (если Git спрашивает про имя ветки):
    ```bash
    git branch -M main
    ```
14. Подключи удалённый репозиторий (подставь свой URL из шага 7 вместо `https://github.com/ТВОЙ_ЛОГИН/founders-vision-tool.git`):
    ```bash
    git remote add origin https://github.com/ТВОЙ_ЛОГИН/founders-vision-tool.git
    ```
15. Отправь код на GitHub (команда ниже). При запросе логина и пароля **пароль от аккаунта GitHub подставлять нельзя** — GitHub больше не принимает обычный пароль. Нужен **токен** (Personal Access Token). Откуда его взять и что вводить — в блоке **7.1.1** ниже.
    ```bash
    git push -u origin main
    ```

**Если папка `founders-vision-tool` лежит внутри другого репозитория (например `cursor`):** используй подготовленную копию. В терминале:
```bash
cd ~/cursor/founders-vision-tool-deploy
git push -u origin main
```
Когда запросит **Username** — введи свой логин GitHub (например `cherr19`). Когда запросит **Password** — вставляй **токен** из шагов 7.1.1 (не пароль от входа в GitHub).

---

#### 7.1.1 Токен GitHub: откуда взять и что вводить при push

GitHub не принимает обычный пароль при `git push`. Нужен **Personal Access Token** — его ты создаёшь в настройках и один раз вставляешь в Терминал вместо пароля.

**Где создать токен (по шагам):**

1. Открой в браузере **[github.com](https://github.com)** и войди в аккаунт.
2. В **правом верхнем углу** нажми на свой **аватар** (фото или иконка) → в меню выбери **Settings**.
3. В левой колонке страницы настроек **в самом низу** нажми **Developer settings**.
4. В левой колонке выбери **Personal access tokens** → **Tokens (classic)**. (Если видишь только Fine-grained tokens — можно использовать их, тогда при создании выбери репозиторий и права **Contents: Read and write**.)
5. Нажми зелёную кнопку **Generate new token** → **Generate new token (classic)**.
6. На странице создания токена:
   - **Note** — любое название, например: `stations deploy`.
   - **Expiration** — выбери срок (например 90 days или No expiration).
   - В блоке **Scopes** поставь галочку **repo** (полный доступ к репозиториям).
7. Внизу нажми **Generate token**.
8. **Скопируй показанный токен сразу** — он больше не отобразится. Выглядит как длинная строка букв и цифр (например `ghp_xxxxxxxxxxxx`).

**Как использовать при push:**

9. В Терминале выполни `git push -u origin main` (из папки проекта или из `founders-vision-tool-deploy`).
10. Когда появится строка **Username for 'https://github.com':** — введи свой логин GitHub (например `cherr19`) и Enter.
11. Когда появится **Password for 'https://ТВОЙ_ЛОГИН@github.com':** — **вставь скопированный токен** (Ctrl+V или Cmd+V) и Enter. **Не вводи пароль от входа в GitHub** — только этот токен.
12. После успешного push код окажется в репозитории на GitHub.

Если токен потерялся — создай новый в том же разделе (Developer settings → Personal access tokens) и используй его при следующем push.

---

### 7.2. Выложить приложение на Vercel (получить публичную ссылку)

Когда проект уже в GitHub (шаг 7.1), подключаем его к Vercel и деплоим.

1. Зайди на **[vercel.com](https://vercel.com)** и нажми **Sign Up** или **Log in**.
2. Выбери **Continue with GitHub** и разреши доступ Vercel к твоим репозиториям.
3. На главной странице Vercel нажми **Add New…** → **Project**.
4. В списке репозиториев найди нужный репозиторий (например **stations** или **founders-vision-tool** — как ты назвал при создании) и нажми **Import** рядом с ним.
5. На странице настроек проекта:
   - **Root Directory** — оставь пустым (репозиторий у тебя уже только с этим приложением). Если по какой-то причине импортировала весь репо `cursor`, укажи **Root Directory**: `founders-vision-tool`.
   - Раскрой **Environment Variables**. Добавь переменные (те же, что в `.env.local`):
     - **Name:** `VITE_SUPABASE_URL` → **Value:** твой URL Supabase (например `https://xxxxx.supabase.co`).
     - **Name:** `VITE_SUPABASE_ANON_KEY` → **Value:** твой Publishable-ключ из Supabase.
     - **(Опционально, для песочницы с Claude)** **Name:** `VITE_OPENROUTER_API_KEY` → **Value:** твой ключ с [openrouter.ai](https://openrouter.ai). Без него приложение работает, но справа не будет панели «обсуждение с Claude».
6. Нажми **Deploy**. Дождись окончания сборки (обычно 1–2 минуты).
7. Когда статус станет **Ready**, на странице проекта будет ссылка вида `https://founders-vision-tool-xxx.vercel.app` — **это ваш общий адрес**. Ты и Алена открываете приложение только по этой ссылке (с разных устройств).

---

### 7.3. Создать комнату и отправить ссылку Алене

1. Открой приложение по **публичной ссылке** (твой адрес на Vercel из шага 7.2).
2. На экране выбора пользователя уже подставлена **комната**. Если нужно — нажми «Создать новую».
3. Нажми **«Ссылка для Алены»** — в буфер скопируется ссылка на твой сайт (например `https://твой-проект.vercel.app/?room=...&user=alena`). **Эту ссылку** отправляй Алене — она откроет её со своего устройства.

### 7.4. Заполнить свои ответы (Таня)

1. На том же экране нажми карточку **«Таня»** (или открой в браузере ссылку с `user=tanya`, если сохраняла).
2. Появится экран «Привет, Таня». Нажми **«Начать аудит»**.
3. Заполни все три части опросника. Ответы сохраняются автоматически в Supabase. Внизу можно нажать «Следующая часть», в конце третьей части — «Перейти к сравнению».

### 7.5. Алена заполняет свои ответы

1. Алена открывает присланную ссылку (с `user=alena`) **со своего устройства**.
2. У неё сразу экран «Привет, Алена» — нажимает **«Начать аудит»**.
3. Заполняет три части. Её ответы пишутся в Supabase; твои она не видит.

### 7.6. Посмотреть сравнение

1. Когда вы обе заполнили опросник, кто угодно (ты или Алена) открывает приложение по **той же публичной ссылке** в той же комнате (тот же `room=...` в URL) и нажимает **«Перейти к сравнению»** (на последней, третьей части формы).
2. Загружаются оба набора ответов, показываются баллы, таблица сравнения и рекомендация (GO / GO с оговорками / PIVOT / NO-GO).
3. При необходимости можно нажать **«Скачать полный отчёт»** (JSON).

### 7.7. Проверить, что данные в Supabase

1. Зайди в [Supabase](https://supabase.com) → свой проект.
2. В левом меню открой **Table Editor** → выбери таблицу **`vision_rooms`**.
3. После сохранения ответов в таблице должны быть строки с полями `room_id`, `tanya_data` и/или `alena_data`, `updated_at`.

### 7.8. Проверить локально (по желанию)

Если хочешь перед деплоем открыть приложение у себя на компьютере: в терминале выполни `cd ~/cursor/founders-vision-tool` и `npm run dev`, открой в браузере `http://localhost:5173`. Это только для проверки; для работы с Аленой с разных устройств нужна публичная ссылка из шага 7.2.

---

Итог: ты и Алена пользуетесь одной комнатой (одна ссылка с одним `room=...`) по **одной публичной ссылке** с разных устройств; данные хранятся в облаке Supabase.
