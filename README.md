# Founders Vision Alignment Tool

React-приложение для независимого заполнения опросника двумя со-основателями и сравнения ответов с подсчётом совместимости (GO / GO с оговорками / PIVOT / NO-GO).

## Запуск

```bash
npm install
npm run dev
```

Откройте в браузере указанный URL (например `http://localhost:5173`).

## Облачное хранилище (Supabase или Firebase)

Чтобы данные синхронизировались между устройствами, настрой **Supabase** (рекомендуется) или Firebase.

**Supabase (пошагово):** открой файл **[SUPABASE_SETUP.md](./SUPABASE_SETUP.md)** в этой папке — там пошаговая инструкция: что открыть в Supabase, как создать таблицу, как взять URL и ключ, что вписать в `.env.local`.

**Firebase:** создай проект в [Firebase Console](https://console.firebase.google.com), включи Firestore, добавь в `.env.local` переменные `VITE_FIREBASE_API_KEY`, `VITE_FIREBASE_PROJECT_ID`, `VITE_FIREBASE_AUTH_DOMAIN` (см. `.env.example`).

Приоритет: если заданы `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY` — используется Supabase; иначе Firebase (если задан); иначе только `localStorage` на текущем устройстве.

## Путь пользователя

**Для Алены (максимально просто):** открыть ссылку (которую отправила Таня, с параметром `?user=alena`) → «Начать аудит» → заполнить три части опросника → «Перейти к сравнению». Ничего подключать не нужно.

**Для Тани:** один раз подключить хранилище (см. ниже), затем такой же путь: открыть ссылку (или свою с `?user=tanya`) → заполнить → сравнение.

На экране выбора пользователя есть кнопки **«Ссылка для Алены»** и **«Ссылка для Тани»** — они копируют в буфер ссылку с нужным `user`, чтобы партнёр сразу попал в свой сценарий без выбора.

## Варианты хранилища (для Тани, один раз)

Подключить нужно что-то одно, чтобы ответы сохранялись в облаке и были доступны с разных устройств.

| Вариант | Что сделать один раз |
|--------|-----------------------|
| **Firebase Firestore** (сейчас в коде) | Создать проект в Firebase, включить Firestore, добавить ключи в `.env.local`, задеплоить приложение (или открывать с того же компьютера). |
| **Supabase** | Создать проект в Supabase, создать таблицу под комнаты/ответы, добавить URL и ключ в env. Код хранилища нужно будет переписать под Supabase API. |
| **Свой бэкенд** (Vercel/Node + БД или JSON) | Написать простой API (сохранение/загрузка по `roomId`), задеплоить вместе с фронтом. Полный контроль, без аккаунтов Google/сторонних сервисов. |

Сейчас в проекте реализован **Firebase**: если заданы переменные `VITE_FIREBASE_*`, данные пишутся в Firestore; иначе используется только `localStorage` (данные только на одном устройстве).

## Как пользоваться (по шагам)

1. **Комната** — при первом открытии в URL подставится `room` (UUID). На экране выбора можно скопировать «Ссылка для Алены» и отправить ей.
2. **Заполнение** — каждый заполняет три части у себя. Ответы автосохраняются. Партнёр видит только статус «уже начала / ещё не начала».
3. **Сравнение** — когда оба готовы, нажать «Перейти к сравнению». Загружаются оба набора, показываются баллы, таблица и рекомендация. Можно скачать полный отчёт (JSON).

## Песочница (чат с Claude)

На экранах «Привет» и «Заполнение» справа отображается панель **Песочница** — чат с Claude Sonnet 4.5 для обсуждения формулировок и мыслей по вопросам (лично для каждого пользователя). Чтобы она работала, в `.env.local` нужно задать ключ OpenRouter:

```bash
VITE_OPENROUTER_API_KEY=sk-or-v1-...
```

Ключ можно получить на [openrouter.ai](https://openrouter.ai). На деплое (Vercel) добавьте ту же переменную в настройках проекта.

## Админ

Только **админ** может создавать новые комнаты и видеть кнопку **«Логи»**. Остальные пользователи открывают приложение по ссылке с уже созданной комнатой (от админа).

**Как войти как админ:** один раз откройте сайт с параметром `?admin=ВАШ_СЕКРЕТ` (например `https://ваш-сайт.vercel.app/?admin=мой_секрет`). Секрет задаётся в `.env.local` и на Vercel: `VITE_ADMIN_SECRET=мой_секрет`. После входа статус админа сохраняется в сессии браузера; кнопка «Логи» отображается в **левом верхнем углу**.

Без комнаты в ссылке не-админ увидит экран «Нет комнаты» и сможет вставить ID комнаты, если он у него есть.

## Логи

Кнопка **«Логи»** (только для админа, в левом верхнем углу) открывает окно с последними действиями приложения. Помогает отлаживать проблемы с синхронизацией или сохранением.
